# Learn

Use this skill when you need to onboard to a library, framework, service, or protocol.

## Workflow

1. Gather sources:
   - Official docs first
   - Version notes/changelogs
   - One or two high-signal examples
2. Synthesize practical output:
   - Core concepts
   - Minimal setup
   - Common patterns
   - Known pitfalls
3. Save result to:
   - `docs/learned/<topic-slug>.md`

## Document format

- `Overview`
- `Quick Start`
- `Core APIs / Patterns`
- `Gotchas`
- `Links`

## Rules

- Prioritize current, primary sources.
- Keep it concise and implementation-focused.
- Include exact versions or dates when behavior is version-sensitive.
